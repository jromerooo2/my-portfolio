Overview

This project implements a VGA-based game entirely in hardware using Verilog. Instead of relying on a CPU or framebuffer, pixel values are generated in real time from combinational logic driven by horizontal and vertical scan counters.

I worked directly with the VGA protocol, including generating sync signals, managing timing constraints, and ensuring the design met pixel clock requirements on FPGA hardware. The system uses simple registers for game state, while all rendering logic is computed on the fly each cycle.

Development involved synthesis and place-and-route using vendor tools, along with debugging timing and signal integrity issues that arise in real hardware designs.

Architecture

VGA Controller

The VGA timing is implemented using a pair of counters that define the raster scan. The system uses configurable timing parameters (typically 800×525 total for 640×480@60Hz) to generate scan coordinates.

Combinational Renderer

Each frame, pixel color is computed from the current scan position and the registered game state. The renderer evaluates paddle, ball, boundary, and background regions using simple comparator logic, with priority encoding selecting the visible pixel.

Game Logic

Ball position, velocity, and paddle positions are stored in registers and updated once per vertical sync pulse, effectively operating at the display refresh rate.

Reflections

This project gave me hands-on experience distinguishing between sequential and structural thinking in hardware design. Instead of just writing step-by-step procedures, I learned how to define a system of logic that operates continuously and in parallel.

Timing was a major learning challenge. I became proficient in using Lattice’s synthesis and place-and-route tools to ensure all combinational paths met the 10 ns pixel clock requirement. This involved adding timing constraints, analyzing critical paths, and iterating on logic depth to reach timing closure at 99 MHz.

Working with the VGA protocol taught me how to follow strict timing requirements, like sync pulses and scan structure, to create a stable 640×480@60Hz display.
