Overview

This project implements a single-cycle RISC-V RV32I processor in SystemVerilog, supporting the full base integer instruction set and all standard instruction formats.

I focused on both datapath design and verification. The CPU was tested using cocotb and simulated with Verilator, allowing me to build a comprehensive testbench that validates instruction behavior, control logic, and edge cases.

This project strengthened my understanding of ISA design, control logic, and hardware verification workflows, as well as the tooling used to simulate and validate RTL designs.

Architecture

The PC register drives the instruction memory address. On each rising clock edge the PC advances by 4 (next sequential instruction) or loads a branch/jump target. Instruction memory is implemented as a synchronous ROM initialized from a .mem file at synthesis time.

A combinational decode block extracts rs1, rs2, rd, funct3, funct7, and the immediate from the 32-bit instruction word. All six RV32I immediate encodings (I, S, B, U, J, and R-type) are sign-extended here.

A 32-bit ALU handles ADD, SUB, SLL, SRL, SRA, AND, OR, XOR, SLT, and SLTU. ALU input A is either rs1 or PC (for AUIPC); input B is either rs2 or the sign-extended immediate. A separate branch comparator evaluates BEQ, BNE, BLT, BGE, BLTU, BGEU.

The data memory supports halfword (LH/SH) and word (LW/SW) accesses with proper sign extension on loads. The writeback mux selects between ALU result, memory read data, PC+4 (for JAL/JALR), and the upper-immediate (for LUI/AUIPC).

Verification

Simulation was driven by a Python cocotb testbench running on top of Verilator. Each test loads a small assembly program into instruction memory, resets the core, and cycles the clock while checking register file state and memory contents against a software reference model.

# cocotb test — verify LW datapath
@cocotb.test()
async def cpu_instr_test(dut):
  cocotb.start_soon(Clock(dut.clk, 1, unit="ns").start())
  await RisingEdge(dut.clk)
  await cpu_reset(dut)

  # LW
  await RisingEdge(dut.clk)
  assert (
    binary_to_hex(dut.regfile.registers[18].value) == "DEADBEEF"
  ), "[CPU] LW datapath failed"

Reflections

This project helped me understand how a CPU really works at a low level. I learned how all the parts (instruction memory, register file, ALU, and data memory) work together in a single cycle, and why that can limit performance.

I also learned how important testing is. Even when I thought my design was correct, automated testing helped me find bugs I hadn’t considered. This showed me that good verification is just as important as writing the design itself.

Moving forward, I want to explore more advanced concepts like pipelining, hazard detection, and performance improvements. This project gave me a strong foundation to build on for more complex computer architecture work.
